# FitCheck (핏체크) — 세컨드핸드/셀렉트숍 사이즈 핏 진단 도구

## 문제
> 세컨드핸드/셀렉트숍 고객이 "이 옷 한 벌이 나한테 맞을까?"를 판단할 수 없어 구매를 망설이는 문제를, AI 핏 진단 + 가상 핏 후기 웹 도구로 해결한다.

- **현재 상태**: 상품당 재고 1개(사이즈 하나)인 구조라 교환이 불가하고, 고객이 핏을 직접 판단해야 함. 상품 상세에 핏 정보가 없어 "나한테 맞을까?" 판단 불가. 사이즈 관련 문의가 전체 CS의 약 30% 차지. 핏 후기도 없어 구매 결정에 참고할 정보가 전무.
- **원하는 상태**: 고객이 키와 몸무게만 입력하면 AI가 쇼핑몰 사이즈표와 비교하여 5단계 핏 판정(작음/약간 작음/적합/약간 큼/큼)을 즉시 제공. 동시에 AI 가상 핏 후기 3-5개가 자동 생성되어 구매 결정을 돕는다.
- **성공 기준**: 핏 진단 사용 고객의 구매 전환율 향상

## 핵심 기능

### 기능 A — AI 핏 진단 (5단계 판정)
- 고객이 **키 + 몸무게만** 입력 (최소 입력으로 진입 장벽 낮춤)
- AI가 쇼핑몰 기존 사이즈표 데이터와 비교하여 체형을 추정
- **5단계 핏 판정**: 작음 / 약간 작음 / 적합 / 약간 큼 / 큼
- 전체 복종 커버: 아우터, 상의, 원피스, 하의(바지/스커트)

### 기능 B — AI 가상 핏 후기 자동 생성 (Clarify로 발견된 신규 기능)
- 상품 실측 정보를 기반으로 AI가 **가상의 핏 후기 3-5개** 자동 생성
- 다양한 체형의 가상 착용자 관점에서 작성 (예: "160cm 55kg인 제가 입으면 상체는 딱 맞고 하체는 여유있어요")
- MVP는 텍스트 후기로 시작, 이미지 생성은 이후 확장

### 기능 C — 상품 핏 레인지 표시 (MD용)
- 쇼핑몰 사이즈표 기반으로 "이 옷은 키 160~168cm인 분에게 적합합니다" 같은 적합 체형 레인지를 자동 계산하여 고객에게 노출

## Clarify 적용 기록

> `/my-clarify` 스킬로 "세컨드핸드/셀렉트숍에서 사이즈 문제 때문에 고객이 구매를 망설이는 걸 해결하고 싶어"를 8개 질문으로 구체화한 결과:

| 항목 | CLAUDE.md (Clarify 전) | Clarify 후 |
|------|----------------------|------------|
| **핏 후기** | 없음 | AI 가상 핏 후기 3-5개 자동 생성 **(신규!)** |
| **판정 단계** | 3단계 (tight/fit/loose) | 5단계 (작음/약간작음/적합/약간큼/큼) |
| **고객 입력** | 키+몸무게+가슴둘레+허리둘레 등 여러 항목 | **키+몸무게만** (AI가 체형 추정) |
| **데이터 소스** | MD가 실측 직접 입력 | **쇼핑몰 기존 사이즈표** 활용 |

## 변화 기록
- **Day 1**: "Claude Code가 뭔지도 몰랐다" → Claude Code 기본 기능 학습, 스킬/MCP 개념 이해
- **Day 3**: "내 업무에 맞는 Clarify 스킬을 직접 만들었다" → 패션 MD 특화 Clarify 스킬(`/my-clarify`) 설계 완료. FitCheck 프로젝트에 Clarify를 적용하여 CLAUDE.md에 없던 **AI 가상 핏 후기** 기능을 발견하고, 고객 입력을 키+몸무게만으로 최소화하는 방향을 확정.
- **가장 크게 달라진 점**: "사이즈 문제 해결하고 싶어"라는 모호한 요청이 Clarify 8개 질문을 거쳐 "5단계 AI 핏 판정 + 가상 핏 후기 자동 생성"이라는 구체적 스펙으로 발전. 특히 Clarify 과정에서 "고객이 실제 구매를 결정할 때 핏 후기가 없다"는 숨겨진 니즈가 드러남. 이것이 Clarify의 가치 — 질문을 통해 본인도 몰랐던 요구사항을 발견하는 것.
