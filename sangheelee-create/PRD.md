# APAC 리더의 AI 업무 자동화 파이프라인

## 문제
> APAC GM이 매주 수동으로 처리하는 AI 트렌드 수집·보고·공유 업무를, 명령 한 줄로 자동화한다.

- **현재 상태**: 매주 AI 뉴스 수집에 30분 이상, 보고서 작성에 1~2시간, 팀 공유까지 총 3시간 이상 수작업 소요. 월간 CS 보고서도 CSV 파일을 열고 직접 분석·작성하는 데 1시간 이상.
- **원하는 상태**: `/주간리포트` 한 번으로 뉴스 수집 → 정제 → 보고서 → PPT → Slack DM까지 15분 안에 완료. `/월간보고서전체`로 CS 데이터 분석부터 보고서 저장까지 자동 실행.
- **성공 기준**: 주간 보고 준비 시간 3시간 → 15분 이내 단축. 월간 CS 보고서 작성 시간 1시간 → 5분 이내.

## 스킬
| # | 스킬명 | 한 줄 설명 | 상태 |
|---|--------|-----------|------|
| 1 | `/주간리포트` | AI 뉴스 수집부터 Slack DM 전송까지 전체 파이프라인 실행 | ✅ 동작 |
| 2 | `/뉴스정제` | 지난 1주일 AI 트렌드 뉴스 수집 후 정제 파일로 저장 | ✅ 동작 |
| 3 | `/트렌드보고서` | 정제된 뉴스 → 경영진 보고용 마크다운 보고서 생성 | ✅ 동작 |
| 4 | `/트렌드PPT` | 주간 트렌드 보고서 → 발표용 PPTX 자동 생성 | ✅ 동작 |
| 5 | `/보고서DM` | 완성된 보고서를 Slack DM으로 지정 멤버에게 전송 | ✅ 동작 |
| 6 | `/월간보고서전체` | CS 점수 CSV 자동 탐지 → 분석 → 월간 보고서 저장 전체 실행 | ✅ 동작 |
| 7 | `/meeting-prep` | 오늘 Google Calendar 미팅 참석자 리서치 → Slack 브리핑 | ✅ 동작 |

## 변화 기록
- **Day 1**: "Claude Code가 뭔지 모르겠다. 명령어를 치면 뭔가 되는 것 같다."
- **Day 3**: "반복 업무를 스킬로 만들면 명령 한 줄로 대체할 수 있다. 내 역할은 검토와 의사결정이면 된다."
- **가장 크게 달라진 점**: 도구를 '사용'하는 것에서 도구를 '설계'하는 관점으로 바뀐 것. 매주 반복하는 보고 작업이 코드 없이 스킬로 자동화될 수 있다는 걸 직접 만들어보고 확인했다.
